# 0.1.7 — 2026-06-21

## Changed

- **`water_dataframe_describe` / `water_dataframe_query` `canvas_disabled` path** — the `ctx.fail()` message and the `errors[].when` / `errors[].recovery` contract fields no longer name the operator-set `CANVAS_PROVIDER_TYPE` env var; the agent-facing error now reads `DataCanvas is not enabled on this server instance.` with a recovery hint to use `water_get_series` directly. The `CANVAS_PROVIDER_TYPE=duckdb` hint moves to a `ctx.log.info` line so self-hosters still see it without leaking it to agents ([#8](https://github.com/cyanheads/usgs-water-mcp-server/issues/8))
- **Server-level `instructions`** — the `water_get_series` spillover note drops `set CANVAS_PROVIDER_TYPE=duckdb`, now reading `DataCanvas, queryable via water_dataframe_query when enabled on this server instance` ([#8](https://github.com/cyanheads/usgs-water-mcp-server/issues/8))
