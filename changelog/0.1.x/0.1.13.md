# 0.1.13 — 2026-07-16

## Fixed

- **`water_get_series` renders every returned value** — `format()` sliced the markdown preview to the last 20 of `result.values`, so `content[]` carried fewer records than `structuredContent.values`. It now renders all values (already bounded upstream to the character-budgeted canvas preview or the last 500 on the no-canvas path), keeping both surfaces in parity. ([#16](https://github.com/cyanheads/usgs-water-mcp-server/issues/16))
- **`water_find_sites` renders `altitude` independent of `drainageArea`** — `altitude` populates in both basic and expanded mode, but was gated behind an expanded-only `drainageArea` check, silently dropping it from `content[]` for basic-mode sites. Each scalar metric (`drainageArea`, `altitude`, `contributingArea`) now renders iff individually present. ([#16](https://github.com/cyanheads/usgs-water-mcp-server/issues/16))
- **`water_find_sites` no longer emits a literal `notice: undefined`** — on the non-truncated path, `notice` was passed into `ctx.enrich()` while `undefined`, and with no `notice.render()` declared the default scalar trailer stringified it as `**notice:** undefined` in `content[]`. `notice` is now spread in only when set, keeping optional enrichment absent from both surfaces. ([#16](https://github.com/cyanheads/usgs-water-mcp-server/issues/16))
- **`water_get_series` spillover notice reports the real preview count** — the DataCanvas-spillover notice hardcoded `SPILLOVER_THRESHOLD` (500), but `spillover()` sizes its preview by a character budget, so the actual count varies. It now interpolates `spillResult.previewRows.length`. ([#19](https://github.com/cyanheads/usgs-water-mcp-server/issues/19))
