# 0.1.11 ‚Äî 2026-07-16

## Added

- **`water_find_sites` `countyCd` query echo** ‚Äî the already-accepted `countyCd` filter now surfaces in the `enrichment.filters` schema, the `ctx.enrich()` payload, and the rendered trailer, matching the other echoed filters (`stateCd`, `siteType`, `bbox`, `huc`) and reaching both `structuredContent` and `content[]` clients. ([#4](https://github.com/cyanheads/usgs-water-mcp-server/issues/4))
- **`usgs-water://site/{siteId}` typed error contract** ‚Äî the resource declares `not_found` (`NotFound`), `invalid_request` (`ValidationError`), and `upstream_error` (`ServiceUnavailable`, retryable) reasons, mapped via the shared `classifyNwisFailure()` + `ctx.fail()` in place of the ad hoc substring-matching catch. A well-formed site number NWIS still refuses now surfaces as typed `invalid_request` rather than an untyped upstream error. ([#21](https://github.com/cyanheads/usgs-water-mcp-server/issues/21))

## Changed

- **`usgs-water://site/{siteId}` `siteId` validation** ‚Äî the param switched from a bare `z.string()` to the shared `SiteNumberSchema` (8‚Äì15 digit pattern), so a malformed site number is rejected at the resource edge ‚Äî with the constraint advertised in the JSON Schema `pattern` ‚Äî instead of consuming a NWIS round-trip. The resource the v0.1.8 edge-validation pass missed. ([#21](https://github.com/cyanheads/usgs-water-mcp-server/issues/21))

## Fixed

- **Site-metadata "available data types / parameters" claim** ‚Äî removed the stale wording from the `usgs-water://site/{siteId}` description, `README.md`, and `docs/design.md`; those per-site availability lists left the runtime shape in v0.1.4 and were never returned by the NWIS site service. ([#7](https://github.com/cyanheads/usgs-water-mcp-server/issues/7))
- **`altitude` mode documentation** ‚Äî corrected from "expanded mode only" to present in both basic and expanded modes when USGS records it, across the `water_find_sites` tool description and `altitude` output `.describe()`, the `NwisSite.altitude` JSDoc, and the `mapSiteRow()` comment. `mapSiteRow()` already extracted `alt_va` unconditionally, so only the docs were wrong; `drainageArea` / `contributingArea` remain genuinely expanded-only. ([#7](https://github.com/cyanheads/usgs-water-mcp-server/issues/7))
