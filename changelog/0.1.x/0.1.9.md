# 0.1.9 — 2026-07-15

## Added

- **`water_get_conditions` `percentileLabel`** — `historicalContext` now carries a plain-language threshold string alongside `percentileClass` (e.g. `"25th–75th percentile"`). The `record-high`/`record-low` labels state they mark percentile-of-record extremes, not verified all-time records, which the enum value alone does not convey. `format()` renders it next to the condition, so `content[]` and `structuredContent` carry the same data. ([#14](https://github.com/cyanheads/usgs-water-mcp-server/issues/14))

## Fixed

- **`water_get_conditions` stat-row selection** — the daily percentile row is now matched against the observation's own calendar date, read from the `YYYY-MM-DD` prefix of the NWIS timestamp via a new `parseObservationDate()` helper, instead of `new Date(currentDateTime)` reprojected into the runtime's local timezone — which could select the neighboring day's percentiles for a reading near midnight. ([#9](https://github.com/cyanheads/usgs-water-mcp-server/issues/9))
- **`hucCd` `.describe()` text** — `water_find_sites` output and `NwisSite.hucCd` no longer claim a fixed 8-digit HUC; NWIS returns 8- and 12-digit codes depending on the level assigned to a site. The description now also states that the `huc` input filter accepts only the first 8 digits — the containing HUC8 subbasin. ([#13](https://github.com/cyanheads/usgs-water-mcp-server/issues/13))
