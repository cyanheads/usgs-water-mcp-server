# 0.1.5 — 2026-06-12

## Added

- **`water_dataframe_query` system-catalog deny** — passes `denySystemCatalogs: true` to the DataCanvas `query()` call, blocking DuckDB system-catalog access from user-supplied SQL.
- **`Dockerfile` healthcheck** — `HEALTHCHECK` probes `/healthz` via a Bun-native `fetch` (the slim runtime ships no curl/wget); adds the `org.opencontainers.image.version` OCI label from a new `APP_VERSION` build arg.
- **`scripts/clean-mcpb.ts`** — post-pack `.mcpb` cleaner wired into `bundle`: runs `mcpb clean`, then strips dependency-shipped agent-doc entries (`node_modules/**/skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach.
- **Packaging and antipattern lint rules** — `lint-packaging.ts` gains bundle-content guards (unanchored-pattern, critical-runtime-path, post-bundle agent-doc) and an identity check that `name`/`title`/`display_name` equal the unscoped package name; `check-framework-antipatterns.ts` gains a `z.coerce.boolean()` env-flag rule.

## Changed

- **`@cyanheads/mcp-ts-core` `^0.9.21` → `^0.10.6`** — framework adoption.
- **Tool error codes** — migrated to the framework's refined `JsonRpcErrorCode` members across `water_find_sites`, `water_get_readings`, `water_get_series`, `water_get_conditions`, and `water_dataframe_query`: validation failures (`invalid_filter`, `invalid_site_format`, `invalid_date_range`, `invalid_sql`) now use `ValidationError`; upstream NWIS 5xx/timeout (`upstream_error`) now uses `ServiceUnavailable`. The `data.reason` contract is unchanged.
- **Server identity** — `createApp()` sets `name` and `title` explicitly to `usgs-water-mcp-server`.
- **`bundle` script** — builds with `bun run build` and runs the new `clean-mcpb.ts` post-pack step.
- **`.mcpbignore`** — root dev-dir patterns anchored with a leading `/` so they no longer also strip nested runtime paths under `node_modules/`.
