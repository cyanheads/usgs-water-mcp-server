# 0.1.4 — 2026-06-08

## Added

- **`water_find_sites` result cap** — responses are now capped at 500 sites; `truncated: true` and `upstreamTotal` (full upstream count) signal overflow ([#6](https://github.com/cyanheads/usgs-water-mcp-server/issues/6))
- **`water_find_sites` expanded drainage fields** — `drainageArea`, `altitude`, and `contributingArea` mapped from `drain_area_va`, `alt_va`, `contrib_drain_area_va` RDB columns when `siteOutput="expanded"` ([#7](https://github.com/cyanheads/usgs-water-mcp-server/issues/7))

## Changed

- **`water_find_sites` output schema** — `dataTypes` and `parameterCds` removed (always empty in standard NWIS mode; callers should use `parameterCd`/`hasDataTypeCd` input filters instead) ([#7](https://github.com/cyanheads/usgs-water-mcp-server/issues/7))

## Fixed

- **`water_get_series` date validation** — `startDate`/`endDate` now validated as `YYYY-MM-DD` format (Zod `.regex()` propagates `pattern` to JSON Schema) and as real calendar dates before the order check; malformed dates previously surfaced as a misleading date-ordering error ([#5](https://github.com/cyanheads/usgs-water-mcp-server/issues/5))
