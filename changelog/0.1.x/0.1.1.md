# 0.1.1 — 2026-06-05

Initial public release of `@cyanheads/usgs-water-mcp-server`. Provides 7 MCP tools and 2 resources over the [USGS National Water Information System (NWIS)](https://waterservices.usgs.gov/) — real-time streamflow, gage height, groundwater depth, water-quality observations, and period-of-record percentile statistics from ~8,000 USGS monitoring sites, with no API key required.

## Added

- **`water_list_parameters`** — static lookup of well-known USGS parameter codes (discharge, gage height, temperature, groundwater depth, and more) with names, units, and thematic domain; no network call
- **`water_find_sites`** — discover USGS monitoring sites by bounding box, state code, FIPS county code, or HUC watershed code; filter by site type (`ST`, `GW`, `LK`, `SP`, etc.) and parameter or data-type availability; returns site number, name, coordinates, state/county/HUC, and available data types
- **`water_get_readings`** — fetch the latest instantaneous (~15 min) values for up to 100 USGS sites in one call; configurable lookback via ISO 8601 duration; surfaces provisional vs. approved qualifiers per reading
- **`water_get_series`** — fetch a historical time series (daily `dv` or instantaneous `iv`) for a site and parameter over a date range; large ranges (>500 records) spill automatically to a DuckDB-backed DataCanvas when `CANVAS_PROVIDER_TYPE=duckdb`; always reports `totalRecords` and `truncated` so callers know when a preview is incomplete
- **`water_get_conditions`** — fetch the current reading alongside the full period-of-record percentile table; classifies the reading as `record-high`, `above-normal`, `normal`, `below-normal`, `low`, or `record-low`; degrades gracefully to `historicalContext: null` when stat data is unavailable
- **`water_dataframe_describe`** — list tables and columns staged on a DataCanvas by `water_get_series`; requires `CANVAS_PROVIDER_TYPE=duckdb`
- **`water_dataframe_query`** — run read-only SQL SELECT against time-series tables staged by `water_get_series`; results capped at 10,000 rows; requires `CANVAS_PROVIDER_TYPE=duckdb`
- **`usgs-water://site/{siteId}`** resource — site metadata (name, coordinates, type, HUC watershed) for a USGS monitoring site by site number
- **`usgs-water://parameters`** resource — full static parameter code catalog as JSON
- **DataCanvas spillover** (`CANVAS_PROVIDER_TYPE=duckdb`) — `water_get_series` materializes large result sets as named DuckDB tables for follow-up SQL; canvas tokens can be chained across calls

## Changed

- **`Dockerfile`** — copies `node_modules` from the build stage so `@duckdb/node-api`'s pre-compiled native binary is preserved in the production image without reinstalling build tools (`python3`, `make`, `g++`)
- **`CANVAS_PROVIDER_TYPE`** — optional env var; without it the server runs in non-canvas mode and `water_get_series` returns a truncated 500-record preview with `truncated: true`

## Security

- **Error-body cap** — upstream NWIS error response bodies are now consumed via `.slice(0, 4096)` before HTML parsing, bounding memory usage on large NWIS error pages
- **URL omitted from error context** — request URLs are no longer forwarded into `serviceUnavailable` / `validationError` error data objects, preventing potential URL-based information leakage through structured error responses
