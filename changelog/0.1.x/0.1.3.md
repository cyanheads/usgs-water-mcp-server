# 0.1.3 — 2026-06-06

## Changed

- **`water_find_sites` output schema** — `stateCd` and `countyCd` are now `optional`; both fields are absent (not empty strings) when `siteOutput="basic"` ([#1](https://github.com/cyanheads/usgs-water-mcp-server/issues/1))
- **`water_dataframe_describe` / `water_dataframe_query` descriptions** — removed `CANVAS_PROVIDER_TYPE=duckdb` env var reference; replaced with "Requires DataCanvas to be enabled on this server instance" ([#2](https://github.com/cyanheads/usgs-water-mcp-server/issues/2))
- **`water_get_series` description and truncation notice** — env var reference removed; plain-language equivalent used instead ([#2](https://github.com/cyanheads/usgs-water-mcp-server/issues/2))
- **`water_get_readings` / `water_get_conditions` error contract** — empty-array path now throws `no_data_for_parameter` (was `site_not_found`); error message notes NWIS returns the same empty response for unknown sites and valid sites with no data for the parameter; `site_not_found` removed from both contracts ([#3](https://github.com/cyanheads/usgs-water-mcp-server/issues/3))

## Added

- **`water_find_sites` enrichment block** — `ctx.enrich({ filters })` populates `structuredContent` with the applied filters (stateCd, siteType, parameterCd, bbox, huc, hasDataTypeCd, siteOutput); rendered as `**Filters applied:** …` in text via `enrichmentTrailer` ([#4](https://github.com/cyanheads/usgs-water-mcp-server/issues/4))
- **`water_get_readings` enrichment block** — `ctx.enrich({ query })` populates sites, parameterCd, and period; rendered as `**Query:** …` ([#4](https://github.com/cyanheads/usgs-water-mcp-server/issues/4))
- **`water_get_series` enrichment block** — `ctx.enrich({ query, notice })` populates site, parameterCd, startDate, endDate, seriesType, and an optional truncation notice on both canvas and no-canvas paths ([#4](https://github.com/cyanheads/usgs-water-mcp-server/issues/4))
