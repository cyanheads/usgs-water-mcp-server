# 0.1.12 — 2026-07-16

## Added

- **`water_find_sites` DataCanvas spillover** — when a query exceeds the 500-site inline cap and `CANVAS_PROVIDER_TYPE=duckdb` is set, the full match set stages to a DuckDB-backed canvas table and the response adds `canvas_id` and `table_name`, so every site past the cap is retrievable via `water_dataframe_query` — a gap-free path that narrowing filters alone cannot guarantee. Mirrors the spillover `water_get_series` already ships. ([#6](https://github.com/cyanheads/usgs-water-mcp-server/issues/6))
- **`water_find_sites` optional `canvas_id` input** — stage into an existing canvas instead of creating a fresh one; applies only when the result is truncated and DataCanvas is enabled. ([#6](https://github.com/cyanheads/usgs-water-mcp-server/issues/6))

## Changed

- **`water_find_sites` overflow signaling** — the truncation notice and `format()` output now point to the staged canvas (`canvas_id` / `table_name`) when DataCanvas is enabled. With no canvas provider, behavior is unchanged: the inline 500-site cap plus `truncated` / `upstreamTotal` and a narrow-your-filters notice. ([#6](https://github.com/cyanheads/usgs-water-mcp-server/issues/6))
